Considered minification impact on code maintainability
Great question! The answer is: no, minification won't affect your source code at all.

Here's how the modern build workflow works:

Source vs. Production
Source files (what you edit): index.html, style.css, script.js remain fully readable
Build output (minified): Gets created in dist/ folder—this is what you deploy to production
You never edit minified files—they're auto-generated
Workflow
Edit style.css and script.js normally (full, readable)
Run npm run build
Get optimized files in dist/ for deployment
Keep editing your source files freely
Benefits
Best of Both Worlds: Readable source code + fast loading for users
No Loss: Original files stay clean and editable
Source Maps (optional): Can add them back to debug production issues
Your Current Situation
Since you prefer maintainability, I'd suggest:

Option 1: Skip minification entirely—keep the site as-is. It already loads fine and is fully editable.

Option 2: Use Parcel's build for development convenience only (bundling, local server) without aggressive minification.

Option 3: Minify only images and enable lazy loading (already done)—skip JS/CSS minification.

Which approach fits your workflow best? Most nonprofit/small sites prioritize maintainability over micro-optimizations anyway.
